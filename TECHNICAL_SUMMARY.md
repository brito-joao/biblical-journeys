# Biblical Journeys - Technical Architecture & Data Guide

## Core Application Overview
The "Biblical Journeys" interactive map is a Next.js (React) static application designed to render historical paths and cities on a 3D Earth using `react-globe.gl` (Three.js wrapper). The aesthetic mirrors a tactile historical explorer's map ("Parchment & Ink").

The core logic handles:
1. Translating geographic coordinates (latitude, longitude) into 3D mapped points.
2. Drawing paths (exodus line) sequentially filtering by active timeline events.
3. Automatically resolving localized string data (`UI_TRANSLATIONS` and object translations).
4. Fetching scripture passages on-demand through `bible-api.com`.

## Internationalization (i18n) Framework
The app is built to be flipped seamlessly between **English (en)** and **Portuguese (pt)**.

### `LocalizedString` Implementation
In `src/types/index.ts`, core textual definitions are typed as `LocalizedString`, representing strings that can strictly take a JSON format mapping.

```typescript
export type LocalizedString = string | { en: string; pt: string };
```

If you are expanding the databases (e.g. `data/journeys/jesus.ts`), you can provide translations directly inside the objects instead of keeping them as bare strings:
```javascript
// BEFORE
{
    city: "Nazareth",
    description: "Mary and Joseph travel to Bethlehem."
}

// AFTER
{
    city: { en: "Nazareth", pt: "Nazaré" },
    description: { 
        en: "Mary and Joseph travel to Bethlehem.", 
        pt: "Maria e José viajam para Belém." 
    }
}
```
The App UI automatically parses these using the central `tData(data.city, language)` helper located in `<TimelineScrubber>` and `<EventCard>`.

### Scripture API (`src/hooks/useBiblePassage.ts`)
We use the free REST API `bible-api.com` which takes a standard verse reference and a translation ID parameter:
- **`web`** (World English Bible) is used for `en`.
- **`almeida`** (João Ferreira de Almeida) is used for `pt`.

The translation is dynamically loaded into `<EventCard>` using the `useLanguage` hook which swaps the ID appropriately. Passages drop into a scrollable modal container automatically.

## How to Add New Maps / Journeys
Data is stored independently per journey inside `src/data/journeys/*.ts`.
Adding a journey simply requires defining `TimelineEvent` records possessing geographic coordinates.

```typescript
export const myNewJourney: TimelineEvent[] = [
    {
        order: 1,
        city: { en: "Rome", pt: "Roma" },
        lat: 41.9028, // Positive is North
        lng: 12.4964, // Positive is East
        year: { en: "c. 60 AD", pt: "c. 60 d.C." },
        description: { 
            en: "Paul arrives in Rome.", 
            pt: "Paulo chega a Roma." 
        },
        verses: "Acts 28:16" // Standard Bible formatting
    }
];
```
Once your array is structured, export it and mount it internally inside `src/data/index.ts` under the `availableJourneys` list.

## Graphic / Mapping Systems
- **`react-globe.gl`**: The core map host component. Renders static map tiles using XYZ format.
- **`TacticalGlobe.tsx`**: Sets `htmlElement` DOM injection points on specific GPS points. The labels themselves are raw HTML elements dynamically generated by `generateGlobeHtmlMarkerDOM.ts`.
- **`GlobeJourneySvgOverlay.tsx`**: Uses DOM portal-weaving to embed SVG lines directly into the `react-globe.gl` DOM structure *behind* the text nodes. Calculates physical angular displacement of nodes against the camera's front-side to cull invisible lines wrapping around the dark side of the globe.

*By adhering to this format, you can confidently inject translated data arrays or hook new map sources scaling seamlessly on both desktop/mobile devices.*
